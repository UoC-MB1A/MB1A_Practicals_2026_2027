    The dataset

    ↑ top

    In this practical, we will first work with a published dataset to learn how to perform various analyses in R. You’ll then be asked to carry out a similar analysis independently with data from a different source.

    For the worked example, we will use data from this Science paper, published in June 2026. This study tracked the migration of a songbird species, the pied flycatcher (Ficedula hypoleuca).

    One of the most surprising findings of this study was that these birds take very different routes depending on the time of year. The authors demonstrated that:

    • In spring, the birds migrate north from their wintering grounds in West Africa. After crossing the Sahara, they stop over near the Mediterranean before continuing to their breeding sites in the north (Fig. 1B).
    • In autumn, the birds take a much longer route. They migrate south from their breeding sites, but — regardless of how far east their breeding site is — they all travel via the Iberian Peninsula (Spain and Portugal) before following the west coast of Africa to their wintering grounds (Fig. 1A). Why they take this detour is currently unknown.
    • As a result, spring migration routes are shorter and more variable than autumn migration routes.

    In our worked example, we will reproduce Figures 1A and 1B from this paper, illustrating this phenomenon.

    The pipe symbol

    ↑ top

    We are going to make copies of these data frames with only the columns we are interested in.

    To do this, we first need to learn about the pipe symbol |>. This is an operator in R which is used to link multiple steps in an analysis.

    For example:

    my_numbers <- c(2, 4, 6, 8)

my_numbers |> sum()
    [1] 20

    Here, we first make a vector of numbers, then we use this vector as the input to the function sum().

    This produces the same result as:

    my_numbers <- c(2, 4, 6, 8)

sum(my_numbers)
    [1] 20

    Pipes make it easier to perform a series of steps on the same variable:

    my_numbers <- c(2, 4, 6, 8)

my_numbers |>
  sum() |>
  sqrt()
    [1] 4.472136

    You can keep adding additional steps to your pipe:

    my_numbers <- c(2, 4, 6, 8)

my_numbers |>
  sum() |>
  sqrt() |>
  round()
    [1] 4

    We can assign the result to a variable using the <- operator as usual.

    my_numbers <- c(2, 4, 6, 8)

my_result <- my_numbers |>
  sum() |>
  sqrt() |>
  round()

print(my_result)
    [1] 4

    Comparison operators

    We can also filter our data frame to just see certain rows.

    To do this, one option is to use comparison operators.

    For example:

    Operator Meaning
    == is equal to
    != is not equal to
    > is greater than
    >= is greater than or equal to
    < is less than
    <= is less than or equal to

    We can use these with single values as well as with data frame columns:

    x <- 3

print(x == 2)
    [1] FALSE

    This shows FALSE because x is not equal to 2.

    x <- 3

print(x == 3)
    [1] TRUE

    This shows TRUE because x is equal to 3.
