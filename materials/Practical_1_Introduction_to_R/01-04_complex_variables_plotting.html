    Packages

    ↑ top

    The standard version of R contains many useful packages and functions for data analysis. However, there are also thousands of additional packages containing additional functions for specific purposes.

    Normally, you need to install these packages in RStudio before you use them.

    Let’s install the package fortunes, to demonstrate.

    install.packages("fortunes")

    library(fortunes)

    Vectors

    ↑ top

    All of the variables we’ve created so far just have a single element - one number, one section of text, or one TRUE/FALSE.

    However, we can also create variables which contain multiple elements.

    One of the most common types of multi-element variable in R is a vector.

    Vectors in R are (usually) specified using the function c.

    my_first_vector <- c(1, 5, 9)
print(my_first_vector)
    [1] 1 5 9

    You can have vectors with all numbers, all text, or all logical values, but everything needs to be the same type.

    character_vec <- c("hello", "this", "is", "a", "character", "vector")
print(character_vec)
    [1] "hello"     "this"      "is"        "a"         "character" "vector"   
    logical_vec <- c(TRUE, FALSE, TRUE)
print(logical_vec)
    [1]  TRUE FALSE  TRUE

    If you have existing variables, you can put them into a vector using their variable names.

    x <- 1
y <- 2
z <- 3

xyz_vec <- c(x, y, z)

    Accessing elements of a vector

    Once our values are in a vector, we can access them using an index, which refers to the position of the value within the vector. Indices are indicated within square brackets - [].

    nice_food <- c("cheese", "beans", "potatoes")

print(nice_food[1])
    [1] "cheese"
    print(nice_food[2])
    [1] "beans"
    print(nice_food[3])
    [1] "potatoes"

    Data Frames

    ↑ top

    Our next variable type is a data frame (sometimes called a tibble), which is one of the types of variable you’ll most commonly use. A data frame is basically a table, consisting of rows and columns. Each row is a single record, corresponding to some entity - a cell, a person, a country - and each column is a specific piece of information.

    For example:

    fruit length weight
    apple 22 150
    pear 30 165
    banana 55 170

    In this data frame, there are three columns - fruit, length and weight, and three rows, each containing information about a specific fruit.

    We can build a data frame using the R function tibble (note that there is also an R function data.frame which you might see referenced elsewhere, usually modern applications use tibble as it is more user-friendly).

    I would specify the data frame above as follows:

    my_data_frame <- tibble(
  fruit = c("apple", "pear", "banana"),
  length = c(22, 30, 55),
  weight = c(150, 165, 170)
)


print(my_data_frame)
    # A tibble: 3 × 3
  fruit  length weight
  <chr>   <dbl>  <dbl>
1 apple      22    150
2 pear       30    165
3 banana     55    170

    Although we can see our data frame using the print() function, it’s not formatted particularly nicely. Instead, we can use the function gt.

    my_data_frame <- tibble(
  fruit = c("apple", "pear", "banana"),
  length = c(22, 30, 55),
  weight = c(150, 165, 170)
)

gt(my_data_frame)
    fruit length weight
    apple 22 150
    pear 30 165
    banana 55 170

    CSV Format

    ↑ top

    The data we will use in many of our practicals is stored in comma-separated values (CSV) format, a plain-text file format where each line represents a row and values in columns are separated by commas.

    For example, for this table:

    fruit length weight
    apple 22 150
    pear 30 165
    banana 55 170

    The contents of the CSV file would be:

    fruit,length,weight
apple,22,150
pear,30,165
banana,55,170

    There is a file provided in your data folder for Practical 1 named weather.csv. Open this file on your computer in a text editor (for example NotePad, TextEdit) and examine it.

    Initiating a Plot

    We will now learn to draw some basic graphs in R, we will learn much more about this over future weeks.

    We start a new, empty plot using the function ggplot. The input for this function is our data frame variable, which contains the data we are going to plot.

    In this example, I’ll plot data from the weather data frame.

    CAM001 2026-07-01 12:00:00 20.1 61 12.7 1016.5 0.0 40 Partly Cloudy W
    CAM001 2026-07-01 18:00:00 18.7 68 10.2 1015.9 0.4 60 Light Rain W

    Adding axes

    We use another function, aes, as an argument to the ggplot function, telling R how various attributes of the data map to the “aesthetics” of the plot.

    For example, here we choose which data we will represent with the x and y axes.

    ggplot(weather, aes(
  x = observation_time,
  y = temperature_c
))

    Adding Data

    Now we can add some data to our plot. In this plotting package, ggplot, we add additional things to our plot as layers, using a + symbol.

    Here, we add a line representing the data using the geom_line function.

    ggplot(weather, aes(
  x = observation_time,
  y = temperature_c
)) +
  geom_line()

    Adding extra features

    We can add some additional features here to make the plot look nicer. For example, here I will change the colour of the line to red, add x and y axis labels and add a title.

    ggplot(weather, aes(
  x = observation_time,
  y = temperature_c
)) +
  geom_line(color = "red") +
  xlab("Date") +
  ylab("Temp (C)") +
  ggtitle("Temperature by Date")
